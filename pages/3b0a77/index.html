C语言操作符优先级

# 前言

表达式的求值顺序是由3个因素决定的：操作符的优先级、操作符的结合性以及操作符是否控制执行的顺序。两个相邻的操作符哪个先执行取决于他们的优先级，如果两者的由优先级相同，那么它们的执行顺序由他们的结核性决定。

# 附：Markdown特殊字符转义

虽然hexo的语法是Markdown格式的，但是在书写时对于特殊字符使用“\“转义是不严谨的做法，最好的方式是使用特殊字符对应的编码，对应如下：

              --> 空格(Space)
!   &#33;           --> 惊叹号 (Exclamation mark) 
”   &#34; &quot;    --> 双引号 (Quotation mark)
#   &#35;           --> 数字标志 (Number sign)
$   &#36;           --> 美元标志(Dollar sign)
%   &#37;           --> 百分号(Percent sign)
&   &#38; &amp;     --> 和号(Ampersand)
|   &#124;          --> 竖线(Vertical bar) 
‘   &#39;           --> 单引号(Apostrophe) 
(   &#40;           --> 小括号左边部分(Left parenthesis) 
)   &#41;           --> 小括号右边部分(Right parenthesis) 
+   &#43;           --> 加号(Plus sign)
-   &#45; &minus;   --> 减号(Minus sign)
*   &#42;           --> 星号(Asterisk)
\   &#92;           --> 反斜杠(Reverse solidus (backslash)) 
<   &#60; &lt;      --> 小于号(Less than) 
=   &#61;           --> 等于符号(Equals sign)
>   &#62;           --> 大于号(Greater than)
?   &#63;           --> 问号(Question mark)
@   &#64;           --> 在标志(Commercial at)
[   &#91;           --> 中括号左边部分(Left square bracket) 
]   &#93;           --> 中括号右边部分(Right square bracket)
{   &#123;          --> 大括号左边部分(Left curly brace) 
}   &#125;          --> 大括号右边部分(Right curly brace)
